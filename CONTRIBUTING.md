# Contributing to DevOps Home Lab

Thanks for your interest in improving this project. This repo is built for **beginners**, so clarity and small fixes are especially valuable.

## First time here?

- **Using the project:** Start with the [README](README.md) and choose your path (Quick Demo, Local Learning, or AWS).
- **Contributing:** You don’t need to be an expert. Fixing typos, clarifying docs, and improving examples all help.

## How to contribute

### Report a bug or unclear docs

1. Open an [Issue](https://github.com/Osomudeya/DevOps-Home-Lab-2026-2027/issues) (use the Bug report or Documentation improvement template if it fits).
2. Use the title to summarize the problem (e.g. “Step 3: command fails on M1 Mac” or “Typo in 02-compose.md”).
3. In the description, include:
   - What you did (e.g. which guide and step).
   - What you expected.
   - What actually happened (error message or screenshot if helpful).
   - Your environment (OS, Docker/Kubernetes version) if relevant.

### Suggest an improvement

Open an issue with the **feature** or **docs** label (if available). Describe the change and why it would help beginners.

### Submit a change (pull request)

1. Fork the repo and create a branch (e.g. `fix/typo-docs-compose` or `docs/improve-prereqs`).
2. Make your changes.
3. Open a **Pull Request** against `main` with:
   - A short title (e.g. “Fix typo in 02-compose.md”).
   - A description of what you changed and why.
4. Link any related issue (e.g. “Fixes #123”).

We especially welcome:

- **Docs and tutorial clarity** (wording, steps, prerequisites).
- **Troubleshooting** (new “common issue” sections or fixes).
- **Accessibility** (clearer headings, links, and code blocks).

## Code and config changes

- Follow existing style (indentation, naming).
- For YAML/JSON/Kubernetes manifests, keep formatting consistent.
- Don’t commit secrets or `.env` files; see [SECURITY.md](SECURITY.md).

## Questions

- Check the [FAQ](docs/local/09-faq.md) and [Troubleshooting](docs/local/08-troubleshooting.md).
- Open a [Discussion](https://github.com/Osomudeya/DevOps-Home-Lab-2026-2027/discussions) or an issue if something is still unclear.

Thank you for helping make this project better for beginners.
